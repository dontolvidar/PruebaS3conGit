Listas, Tuplas y Diccionarios en Python¬∂

Introducci√≥n¬∂

Este es un cuaderno introductorio a las colecciones como tuplas, listas y diccionarios en Python. Para una introducci√≥n m√°s extensa, ir aqu√≠. Empezamos con el concepto de Iterables e iteradores.

Iterables e iteradores¬∂

Hay dos protocolos que es muy probable tenga que utilizar, o posiblemente necesite implementar en alg√∫n momento; estos son el protocolo Iterable y el protocolo Iterador. Estos dos protocolos est√°n estrechamente relacionados y son muy utilizados y respaldados por un gran cantidad de tipos.

Una de las razones por las que los iteradores y los iterables son importados es que se pueden usar con sentencias for en Python; esto hace que sea muy f√°cil integrar un iterable en el c√≥digo que necesita procesar una secuencia de valores a su vez

Iterables¬∂

El protocolo Iterable es utilizado por tipos donde es posible procesar su contenido. uno a la vez.

Un Iterable es un objeto que proporcionar√° un Iterador que puede ser utilizado para realizar este procesamiento.

Como tal, el iterable no es el iterador en s√≠ mismo; sino el proveedor del iterador.

Hay muchos tipos iterables en Python, incluidas listas, conjuntos, diccionarios, tuplas, etc. Todos estos son contenedores iterables que proporcionar√°n un iterador.

Es posible adem√°s construir clases que generen iteradores personalizados. De momento trabajaremos con los tipos que ya lo soportan, mencionados arriba.

Iteradores¬∂

Un iterador es un objeto que devolver√° una secuencia de valores.

Los iteradores pueden ser finitos de longitud o infinito (aunque la mayor√≠a de iteradores orientados a contenedores proporcionan un conjunto fijo de valores.

Generadores¬∂

Un generador es una funci√≥n especial que se puede utilizar para generar una secuencia de valores que se van a iterar sobre demanda (es decir, cuando se necesitan los valores) en lugar de que producido todo por adelantado.

Lo √∫nico que hace que una funci√≥n generadora trabaje como generador es el uso de palabra clave yield.

Ejemplo de una funci√≥n generadora¬∂

def gen_numbers():
    yield 1
    yield 2
    yield 3

Esta es una funci√≥n generadora, ya que tiene al menos una declaraci√≥n yield (de hecho, tiene Tres).

Cada vez que se llama a la funci√≥n gen_numbers () dentro de una instrucci√≥n for devolver√° uno de los valores asociados con una declaraci√≥n yield; en este caso el valor 1, luego el valor 2 y finalmente el valor 3 antes de que regrese (termine).

Ahora construimos una ciclo for que va llamando los n√∫meros a la medida que los necesita. Observe que la variable i en este ejemplos es un iterador entregado por la funci√≥n gen_numbers().

for i in gen_numbers():
    print(i)

Creaci√≥n de una clase iterable¬∂

M√°s adelante veremos clases a plenitud. Aqu√≠ introducimos una clase muy sencilla para mostrar como crear un iterable personalizado.

class Evens(object):
    def __init__(self, limit):
        self.limit = limit
        self.val = 0
    
    # Hace esta clase iterable
    def __iter__(self):
        return self
    
    # Hace esta clase un iterador
    def __next__(self):
        if self.val > self.limit:
            raise StopIteration
        else:
            return_val = self.val
            self.val += 2
            return return_val

Solamente vamos a resaltar unas pocas cosas de esta clase Events

  • El m√©todo __iter__() retorna self; este es un patr√≥n muy com√∫n y asume que la clase tambi√©n implementa el protocolo iterador.

  • El m√©todo __next__() retorna el siguiente valor en la secuencia o dispara una excepci√≥n con StopIteration para indicar que no hay m√°s valores disponibles.

El protocolo implica que cuando Python encuentra la definici√≥n de la funci√≥n __iter__() construye la maquinaria para que la clase sea un iterable. De la misma forma, cuando encuentra la funci√≥n __next__() construye la maquinaria para que la clase entregue un iterador.

Veamos en acci√≥n Evens. Discuta en la clase.

print('¬°Empezamos')
for i in Evens(6):
    print(i,end=', ')
print('\nHecho!')

Colecciones¬∂

Una colecci√≥n es un contenedor de objetos del mismo tipo. Por defecto en Python existen cuatro tipos de contenedores:

  • Tuplas (tuple). Esta es una colecci√≥n de objetos que est√°n ordenados y son inmutables (no pueden modificarse). Las tuplas admiten elementos repetidos y sus miembros son indexados.

  • Listas (list). Son colecciones de objetos que est√°n ordenado y son mutables, es decir, sus contenidos pueden ser modificados. Los elementos son indexados y permite duplicados.

  • Conjuntos (set). Son contenedores de datos que no son ordenados ni indexados. Son mutables, pero no admiten duplicados.

  • Diccionarios (dictionary) Son contenedores no ordenados, que son indexados mediante una clave, la cual referencia a un valor. El valor es retornado cuando se le solicita con la clave. No se admiten claves repetidas, pero si valores repetidos.

Recuerde que todo en Python es realmente un tipo de objeto.

Conclusi√≥n¬∂

Las listas y los diccionarios son los tipos de objetos m√°s usados en Python.

Debido a su diferencia en la forma de accesar los datos, tienen diferentes usos, dependiendo de la situaci√≥n.

M√©todos de los diccionarios¬∂

M√©todo

Descripci√≥n

clear()

Elimina todos los elementos del diccionario.

copy()

Devuelve una copia poco profunda del diccionario.

get(clave[,valor])

Devuelve el valor de la clave. Si no existe, devuelve el valor valor si se indica y si no, None.

items()

Devuelve una vista de los pares clave: valor del diccionario.

keys()

Devuelve una vista de las claves del diccionario.

pop(clave[,valor])

Devuelve el valor del elemento cuya clave es clave y elimina el elemento del diccionario. Si la clave no se encuentra, devuelve valor si se proporciona. Si la clave no se encuentra y no se indica valor, lanza la excepci√≥n KeyError.

popitem()

Devuelve un par (clave, valor) aleatorio del diccionario. Si el diccionario est√° vac√≠o, lanza la excepci√≥n KeyError.

setdefault(clave[,valor])

Si la clave est√° en el diccionario, devuelve su valor. Si no lo est√°, inserta la clave con el valor valor y lo devuelve (si no se especifica valor, por defecto es None).

update(iterable)

Actualiza el diccionario con los pares clave: valor del iterable.

values()

Devuelve una vista de los valores del diccionario.
