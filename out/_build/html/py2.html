Funciones puras¬∂

Las funciones puras tienen dos propiedades.

  • Siempre produce la misma salida para los mismos argumentos. Por ejemplo, 3 + 7 siempre ser√° 10 pase lo que pase.

  • No cambia ni modifica la variable de entrada.

La segunda propiedad tambi√©n se conoce como inmutabilidad. El √∫nico resultado de la funci√≥n pura es el valor que devuelve. Son deterministas. Los programas realizados mediante programaci√≥n funcional son f√°ciles de depurar porque las funciones puras no tienen efectos secundarios ni E / S ocultas. Las funciones puras tambi√©n facilitan la escritura de aplicaciones paralelas / concurrentes. Cuando el c√≥digo est√° escrito en este estilo, un compilador inteligente puede hacer muchas cosas: puede paralelizar las instrucciones, esperar a evaluar los resultados cuando los necesite y memorizar los resultados, ya que los resultados nunca cambian mientras la entrada no cambie.

Revise el siguiente ejemplo. Recibe una lista de n√∫mero y regresa una nueva lista con los valores elevado al cuadrado.

# Ejemplo de una funci√≥n pura

# Una funci√≥n pura que no
# cambia la lista de entrada y
# devuelve la nueva lista

def pure_func(List):
    New_List = []
    for i in List:

    return New_List

Recursi√≥n¬∂

En la programaci√≥n funcional, no existe el concepto de bucle for o bucle while, sino que se utiliza la recursividad. La recursividad es un proceso en el que una funci√≥n se llama a s√≠ misma directa o indirectamente. En el programa recursivo, se proporciona la soluci√≥n al caso base y la soluci√≥n al problema mayor se expresa en t√©rminos de problemas menores. Puede surgir una pregunta ¬øcu√°l es el caso base? El caso base puede considerarse como una condici√≥n que le dice al compilador o int√©rprete que salga de la funci√≥n.

Ejemplo: consideremos un programa que encontrar√° la suma de todos los elementos de una lista sin usar ning√∫n bucle for.

# Ejemplo de recursi√≥n

# Funci√≥n recursiva para encontrar
# suma de una lista

def Sum(L, i, n, count):

    # Caso base
    if n <= i:
        return count

    count += L[i]

    # Proceso recursivo
    count = Sum(L, i + 1, n, count)

    return count

# Ejemplo de uso
L = [1, 2, 3, 4, 5]
count = 0
n = len(L)
print(Sum(L, 0, n, count))

Objetos de primera clase¬∂

Los objetos de primera clase se manejan uniformemente en todo momento. Pueden almacenarse en estructuras de datos, pasarse como argumentos o usarse en estructuras de control. Se dice que un lenguaje de programaci√≥n admite funciones de primera clase si trata las funciones como objetos de primera clase.

  • Puede pasar la funci√≥n como par√°metro a otra funci√≥n.

  • Puede devolver la funci√≥n desde una funci√≥n.

Funciones integradas de orden superior¬∂

Para facilitar mucho el procesamiento de objetos iterables como listas e iteradores, Python ha implementado algunas funciones de orden superior de uso com√∫n. Estas funciones devuelven un iterador que ahorra espacio. Algunas de las funciones integradas de orden superior son:

  • map(): la funci√≥n map() devuelve una lista de los resultados despu√©s de aplicar la funci√≥n dada a cada elemento de un iterable dado (lista, tupla, etc.)

  • filter(): El m√©todo filter() filtra la secuencia dada con la ayuda de una funci√≥n que prueba que cada elemento de la secuencia sea verdadero o no.

Veamos los siguientes dos ejemplos que ilustran map() y filter().

# Ejemplo del uso de map()

# Retorna el doble de n
def addition(n):
    return n + n

# doblamos todos los valores de una lista usando map
numbers = (1, 2, 3, 4)
results = map(addition, numbers)

#  Print no imprime los resultados
# porque se recibe un iterable
print(results)

# Asi recuperamos del iterable los valores e imprimimos
for result in results:
    print(result, end = " ")
